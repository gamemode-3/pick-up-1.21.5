- [curseforge](https://www.curseforge.com/minecraft/mc-mods/pick-up-inventory-management).

## contribute
you are welcome to contribute to this project. here's what you can do:
### open issues
opening issues makes sense for:
- bugs you found
- cross-compatibility issues
- suggestions for new elements that keep the mod straightforward and simple (no big new features)
- version updates (except snapshots / previews)

### contribute code
i'm pretty busy right now so if you can code or want to learn it by contributing, that would help me a lot!

#### contributions should address an issue
before opening a pull request (PR), make sure your contibution addresses an issue that is open and tagged by me. 
otherwise i may not accept your PR because i may not want that change to be implemented.

#### issue assignment
before getting started, make sure nobody is assigned to the issue yet. 
by asking to be assigned you reserve the issue for yourself. 
before asking, please make sure you have the time to resolve the issue in a reasonable time frame (obviously depends on the issue, small fixes shouldn't take longer than a day tho). 
if you take too long i might do it myself. i will, however, communicate that ahead of time.

#### AI policy
tldr: do not contribute AI generated code. you may use single-line AI autocomplete. you should know what every line of your code does and why.

when i say AI here, i mean LLMs. you can, of course, use AI to learn how to execute an idea. it's very useful for that. 
single line autocomplete is fine, so long as it is genuinely only completing what you intended to write anyway.
that means you should have a good understanding of why each line of code is the way it is.
the reasons behind this are: 
- AI systems often copy open source code without attribution, violating their licenses. 
  i respect other projects' licenses and do not want to risk illegitimately having their code in my codebase.
- AI systems often produce misaligned or over-engineered code and you understanding the code means you can catch errors or unnecessary complexity.
- i like humans. we're kinda cool. i think we should continue existing ♡
